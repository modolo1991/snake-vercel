# Snake Deluxe update — 2026-03-28

## What changed
- Rebuilt the game as a richer single-file static site in `index.html`.
- Improved mobile handling:
  - safe-area aware layout (`viewport-fit=cover`, `env(safe-area-inset-*)`)
  - dynamic viewport height fix for iPhone/Safari browser chrome changes
  - larger tap targets and phone-first layout
  - optional fullscreen button where the browser allows it
  - pause-on-tab-hide to reduce accidental deaths when browser UI interrupts play
- Added gameplay depth:
  - persistent coin currency via `localStorage`
  - shop/loadout for skins and power-ups
  - unlockable skins: Mint, Gold, Neon, Coral
  - unlockable power-ups: Shield, Slow Time, Coin Magnet
  - tunnel / hole teleport pairs on later levels
  - multiple levels with increasing speed, hazards, obstacles, and better coin rewards
  - level progression gated by food collected
- Preserved the simple Snake feel:
  - keyboard still works
  - swipe controls still work
  - on-screen direction pad still works

## Persistent data
Stored in browser `localStorage` under:
- `snake-deluxe-save-v1`

It keeps:
- best score
- total coins
- owned items
- equipped skin
- active power-ups

## How to test locally
From the workspace root:

```bash
./start_snake_server.sh 8080
```

Then open one of:
- `http://127.0.0.1:8080/snake.html`
- `http://127.0.0.1:8080/snake-vercel/`

## Suggested test checklist
- iPhone / mobile browser:
  - load page in portrait and landscape
  - confirm content stays visible around notches / bottom bars
  - confirm big arrow buttons are easy to hit
  - swipe on board to steer
  - tap Full screen if offered by browser
- gameplay:
  - collect apples and confirm level-up overlay appears
  - collect coins and confirm total persists after refresh
  - buy a skin and confirm it stays equipped after refresh
  - buy/activate a power-up and confirm it affects the next run
  - enter tunnels and confirm teleporting works
  - crash with Shield active and confirm you get one save

## Remaining limitations / notes
- Mobile Safari does not always allow true fullscreen the way installed PWAs do; this build reduces browser-UI pain where possible, but cannot fully control Safari chrome from a normal webpage.
- This is still a single-page static app with no backend, so save data is per-browser/device.
- No audio yet.
